 SECTION 1: PROJECT OVERVIEW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

SmartTraffic AI is a browser-based AI-powered traffic event management platform
built for Bengaluru city traffic planning. It trains a machine learning model 
LIVE in the browser using the real Bengaluru traffic incident dataset 
("dataset csv file.csv") to:

  • Forecast congestion impact for upcoming events (0–100 severity score)
  • Predict risk levels: Low / Moderate / High / Critical
  • Recommend exact resource deployment (officers, barricades, diversions, etc.)
  • Explain AI decisions using SHAP-style feature contributions
  • Simulate what-if scenarios side-by-side
  • Generate PDF operational reports
  • Provide an AI chatbot for natural-language explanations

IMPORTANT: All intelligence is derived EXCLUSIVELY from the real CSV dataset.
No synthetic data, pre-trained models, or external APIs are ever used.

 SECTION 2: AI MODEL — HOW IT IS TRAINED ON THE CSV
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

FILE USED: "dataset csv file.csv" → copied to public/dataset.csv
           (Both files are identical — 4.5 MB of real Bengaluru incidents)

HOW TRAINING WORKS:
1. The browser fetches /dataset.csv at startup (via SWR + PapaParse)
2. Data is parsed and cleaned (invalid rows, nulls, type mismatches filtered)
3. Records where event_type is "planned" or "unplanned" are used (others skipped)
4. An engineered "impact" score (0–100) is computed from raw CSV columns
5. A multivariate regression model is trained using gradient descent
6. 5-fold cross-validation is run to evaluate generalization
7. Feature importance is computed for explainability (XAI)
8. The trained model is used LIVE for all predictions in the app

TRAINING ALGORITHM: Multivariate Linear Regression with Gradient Descent
  • Learning rate: 0.06
  • Full training epochs: 900
  • CV epochs per fold: 300
  • Cross-validation folds: 5 (k=5)

FEATURE ENGINEERING (9 features derived from CSV columns):
  1. Event Cause Profile    — target encoding of event_cause column
     (Each cause gets its historical average impact as its numeric value)
  2. Corridor Sensitivity   — target encoding of corridor column
     (Each corridor gets its historical average impact)
  3. Zone History           — target encoding of zone column
     (Each zone gets its historical average impact)
  4. Priority Level         — one-hot: High=1.0, Medium=0.5, Low=0.0
     (from the "priority" CSV column)
  5. Road Closure           — binary: 1 if requires_road_closure=TRUE
  6. Planned Event          — binary: 1 if event_type="planned"
  7. Peak Hour              — binary: 1 if hour is 08:00–11:00 or 17:00–20:00
     (derived from start_datetime column)
  8. Incident Duration      — log-scaled minutes between start and resolution
     (derived from start_datetime, resolved_datetime, closed_datetime)
  9. Weekend Effect         — binary: 1 if Saturday or Sunday
     (derived from start_datetime day-of-week)

TARGET VARIABLE (Impact Score 0–100):
  Computed from these CSV columns using domain knowledge:
  • event_cause          → base impact (accident=78, congestion=72, etc.)
  • priority             → +14 (High), +7 (Medium), +0 (Low)
  • requires_road_closure → +12 if TRUE
  • event_type           → +6 if planned
  • corridor             → +8 if on a named corridor
  • duration             → +0 to +18 (log-scaled minutes)
  • hour of day          → +6 during peak hours

RISK BANDS:
  • Low:      0–41  (light congestion, minimal intervention needed)
  • Moderate: 42–61 (notable congestion, standard resource deployment)
  • High:     62–79 (severe congestion, heightened response needed)
  • Critical: 80+   (gridlock risk, maximum resource mobilisation)

MODEL METRICS (approximate, vary per dataset run):
  • Risk-band Accuracy: ~85–92%
  • R-squared (R²):     ~0.75–0.90
  • RMSE:               ~5–12 impact points
  • Cross-validation R²: ~0.70–0.88

CSV COLUMNS USED IN MODEL:
  id, event_type, event_cause, requires_road_closure, start_datetime,
  resolved_datetime, closed_datetime, modified_datetime, status, corridor,
  priority, police_station, zone, latitude, longitude, veh_type

RESOURCE FORMULA:
  officers = max(2, impact * 0.24 * closure_mult * peak_mult)
  marshals  = max(1, impact * 0.12 * peak_mult)
  barricades = max(2, impact * 0.30 * closure_mult)
  where closure_mult = 1.4 if road closure, else 1.0
        peak_mult = 1.2 during peak hours, else 0.85

 SECTION 4: ALL FEATURES OF THE PLATFORM
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1.  LIVE AI TRAINING
    Model trains in the browser on page load. No backend server needed.
    Uses PapaParse to parse CSV, then gradient descent runs client-side.

2.  TRAFFIC CONGESTION FORECASTING
    Predicts a 0–100 impact score for any event configuration.
    Accounts for cause, location, time, duration, road closure.

3.  RISK LEVEL CLASSIFICATION
    Four bands: Low (<42), Moderate (42–61), High (62–79), Critical (80+)
    Color-coded throughout the interface.

4.  RESOURCE RECOMMENDATION ENGINE
    Computes exact counts for: police officers, traffic marshals, rapid response
    units, ambulances, barricades, checkpoints, diversions, lane restrictions,
    emergency corridors. All derived from predicted impact score.

5.  EXPLAINABLE AI (XAI)
    SHAP-style contribution bars show which features pushed the score up/down.
    Feature importance ranking with direction (increases/decreases risk).

6.  24-HOUR CONGESTION TIMELINE
    Hour-by-hour congestion forecast for the entire day, accounting for
    morning and evening peak hours, event timing, and impact severity.

7.  HISTORICAL ANALYTICS
    Full exploratory data analysis of all incidents in the dataset:
    hourly patterns, weekly patterns, seasonal trends, cause distribution,
    corridor rankings, zone rankings, impact histograms, scatter plots.

8.  DIGITAL TWIN VISUALIZATION
    Animated SVG-based corridor visualization showing where to place
    officers, barricades, checkpoints, and diversions on a stylized road.

9.  SCENARIO COMPARISON (WHAT-IF ANALYSIS)
    Compare two different event configurations side-by-side.
    Overlay their 24-hour congestion forecasts on one chart.

10. MOST SIMILAR HISTORICAL INCIDENTS (KNN)
    Finds the top 5 nearest-neighbour matches from the dataset using
    weighted categorical similarity (cause, corridor, zone, priority,
    closure, hour). Shows their actual recorded impact.

11. CORRELATION HEATMAP
    Pearson correlation matrix for 7 engineered features,
    color-coded from teal (negative) to orange (positive).

12. SMART ALERTS SYSTEM
    Automatically shows context-specific warnings:
    - Critical/High risk alerts when impact score is severe
    - Resource intensity warnings when large deployments are needed
    - Diversion recommendation alerts

13. DEPLOYMENT CHECKLIST
    Auto-generated ordered action checklist derived from the resource plan.
    Additional steps added for High/Critical scenarios.

14. PDF REPORT EXPORT
    Full operational report with all model data, scenarios, and
    recommendations. Generated client-side using jsPDF.

15. AI ASSISTANT CHATBOT
    Natural-language question answering grounded in real model statistics.
    Answers questions about congestion, officers, factors, zones, risk scores.

16. DATASET EXPLORER
    Interactive searchable table showing raw CSV data with column headers.
    Filter by any text across 9 columns in real-time.

17. DATA HEALTH SCORING
    Quantifies data quality (0–100) based on missing cell percentages
    across key traffic fields. Penalizes missingness, rewards completeness.

18. MOBILE-RESPONSIVE NAVIGATION
    Collapsible mobile sidebar navigation for smaller screens.

19. DARK MODE DESIGN
    Full dark-mode-only premium UI with glassmorphism cards, neon accents,
    OKLCH color space colors, custom scrollbars, and micro-animations.

20. LIVE MODEL STATS IN SIDEBAR
    Sidebar footer shows real-time accuracy % and R² from the trained model.
